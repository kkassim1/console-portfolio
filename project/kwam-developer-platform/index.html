PROJECT 01 / PLATFORM ENGINEERING

Kwam Developer Platform

A paved road from idea to observable service.

Kwam Developer Platform is a working, local-first internal developer platform prototype that gives Go developers a standardized golden path for creating services. Its CLI generates services with health and readiness endpoints, basic Prometheus metrics, structured logging, a test entrypoint, non-root containers, and ownership metadata.

The platform repository includes automated quality and container-security gates, local Docker Compose and Prometheus observability, and an opt-in AWS reference architecture built with OpenTofu, GitHub OIDC, ECR, and ECS/Fargate.

CAPABILITIES

What I built

  • Self-service Go CLI with embedded, versioned service templates
  • Secure-by-default, non-root container workflow
  • Standard health, readiness, metrics, logging, and ownership contracts
  • GitHub Actions quality gates and container vulnerability scanning
  • Local Docker Compose runtime and Prometheus observability
  • Opt-in, cost-aware AWS reference architecture
  • Protected deploy, health verification, rollback, and cleanup workflow
  • Developer, operational, cost-safety, and architecture documentation

CURRENT STATE

Project status

The local developer workflow has been tested end to end, including service generation, automated tests, endpoint contracts, container builds, Docker health checks, non-root execution, and local observability.

The AWS reference architecture and protected deployment workflow are implemented and statically validated but have not been applied to an AWS account. Therefore, I describe the project as deployment-ready rather than currently cloud-hosted.

Reusable CI and deployment workflows for standalone generated repositories, along with existing-application validation and registration, are the next milestones.
